# Steam 商店页材料准备包

## 文档目的

本文档用于准备 `Lucky Dog Rise` 的 Steam 商店页后续填写材料，包括文本、图片、截图、视频、动图、内容调查与审核前检查项。

本文档不替代 Steamworks 后台，以后台实际清单和 Steam 官方文档为准。

## 后续文本材料

### 短描述

用途：

- 显示在商店页首屏右侧简介位置。
- 也可能出现在 Steam 搜索、推荐列表和其他空间受限的位置。

填写原则：

- 只写纯文本，不使用换行、项目符号或富文本格式。
- 不写发行日期、即将推出、现在可玩等时间敏感文字。
- 先讲清楚产品是什么，再讲关键差异点。

建议英文版本：

```text
A cozy desktop dog companion that reacts as you type, click, work, and play. Earn chips, open blind boxes, collect outfits, and switch into quick video poker breaks whenever your lucky pup is ready.
```

建议简体中文版本：

```text
一只会陪在桌面上的幸运小狗，会在玩家打字、点击、工作和游玩时作出反应。赚取筹码、开启盲盒、收集装扮，也可以随时切入一局轻松的视频扑克小休息。
```

- 使用清晰小标题，不堆长段落。

### 内容调查说明

建议填写口径：

- 游戏包含视频扑克规则，但不涉及真实货币博彩。
- 不应使用 `gambling` 作为核心卖点。
- 若后台内容调查询问博彩/赌博，应诚实说明存在虚拟筹码扑克玩法，但不可兑换现金、奖品或真实货币价值。
- 当前不涉及成人内容、血腥暴力、药物、恐怖、仇恨或在线多人互动。

需要主人确认：

- @主人 [请确认首发版本是否完全不含真实货币购买、付费抽取、可交易库存或现金价值奖励。]
- @主人 [请确认是否公开展示盲盒掉落概率；如展示，需要以 Luban 表最终配置为准。]

### 发行日期文案

建议策略：

- 商店页早期先用较模糊的玩家可见日期，例如 `Coming Soon` 或年份/季度。
- 后台真实目标日期可以先按开发排期保守填写，后续再调整。
- 不在短描述或胶囊图中写日期，避免过期。

需要主人确认：

- @主人 [请确认玩家可见发行日期使用 Coming Soon、2026、2026 Q4、月份，还是具体日期。]

## 图片素材清单

### 商店胶囊图

必须准备：

- Header Capsule：`920 x 430`
- Small Capsule：`462 x 174`
- Main Capsule：`1232 x 706`
- Vertical Capsule：`748 x 896`

制作原则：

- 只能包含游戏美术、游戏名称和官方副标题。
- 不放评测分数、奖项、折扣文案、宣传语、二维码、网址或其他营销文字。
- 游戏 logo 必须在小尺寸下仍然清晰。
- 画面应直接传达“桌面小狗 + 轻松可爱 + 卡牌/筹码”的产品气质。

建议画面方向：

- 小狗位于画面主视觉，带桌面窗口或任务栏暗示。
- 筹码、卡牌、盲盒作为辅助元素，不要喧宾夺主。
- Logo 使用统一字形，保持所有胶囊图一致。

### 截图

必须准备至少 5 张。

官方要求与建议：

- 最低 `1920 x 1080`。
- `16:9` 比例。
- 截图应展示真实 gameplay。
- 不使用概念图、预渲染宣传图、奖项、营销文案或大段说明文字。
- 菜单截图只有在菜单本身是独特玩法组成时才建议使用。

建议截图列表：

1. 桌宠模式：小狗在桌面/任务栏附近陪伴，展示核心第一印象。
2. 打字或点击反馈：小狗根据输入反应，展示桌面互动。
3. 视频扑克模式：显示手牌、筹码、赔率表或结算反馈。
4. 盲盒开启：展示开盒表演和奖励获得。
5. 装扮/背包：展示收集、装备和小狗外观变化。
6. 可选：设置面板中的玩家可见设置，不展示 Debug 页。

### 页面背景

可选准备：

- Page Background：`1438 x 810`

制作原则：

- 背景应偏氛围，不抢商店页正文内容。
- 如果不上传，Steam 会用最后一张截图自动生成背景。

建议：

- 可以先不做，等主视觉和截图完成后再决定。

### 社区与客户端图标

必须准备：

- Shortcut Icon：`256 x 256`，`.ico` 或 `.png`
- App Icon：`184 x 184`，`.jpg`

建议：

- 使用小狗头部、幸运符号或简化 logo。
- 小尺寸下不要放复杂文字。

### 库资产

必须准备：

- Library Capsule：`600 x 900`
- Library Hero：`3840 x 1240`，`.png`
- Library Logo：`1280` 像素宽或 `720` 像素高，`.png` 透明背景
- Library Header Capsule：`920 x 430`

制作原则：

- Library Capsule 必须包含游戏 logo，不放其他文字。
- Library Hero 只放美术，不放任何文字。
- Library Logo 只放游戏标题字标，可带简单图标，透明背景。

## 视频与动图

### 商店预告片

Steam 要求发行流程中上传至少一个预告片。

官方技术建议：

- 最高使用 `1920 x 1080`。
- `30/29.97 fps` 或 `60/59.94 fps`。
- 高码率，建议 `5000 Kbps` 以上。
- 容器使用 `.mov`、`.wmv` 或 `.mp4`。
- 优先 `H.264` 视频与 `AAC` 音频。
- 推荐 `16:9`。

内容建议：

- 第一个预告片应以 gameplay 为主，因为 Steam 用户优先想看实际玩法。
- 前 10 秒必须让玩家明白游戏是什么。
- 即使静音播放，也应能看懂核心卖点。
- 保留游戏 HUD/UI，帮助玩家理解如何互动。

建议预告片结构：

1. 0-3 秒：小狗出现在桌面，形成第一眼记忆点。
2. 3-8 秒：打字/点击，小狗反应，筹码增长。
3. 8-14 秒：视频扑克下注、保留、抽牌、结算。
4. 14-20 秒：盲盒开启，获得道具。
5. 20-26 秒：背包装扮，小狗外观变化。
6. 26-30 秒：Logo + Steam Wishlist 结尾画面。

注意：

- Steam 会根据第一个可见预告片自动生成 6 秒 microtrailer，无法自定义。
- 因此第一个预告片中应尽量让各时间点都有清晰 gameplay，不要前半段全是 logo 或黑场。

### 关于此游戏中的动图

官方允许在“关于此游戏”中嵌入截图或循环 GIF，用于突出核心特色。

限制与建议：

- 单个图片或 GIF 应小于 `5MB`。
- 关于此游戏区域的截图和 GIF 总文件体积建议低于 `15MB`。
- 动图尽量不含文字；如果含文字，后续本地化时需要为每个语言重新上传。

建议动图数量：

- 首轮准备 3 个短循环 GIF 即可。

建议动图内容：

1. 桌宠反应：3-5 秒，打字/点击触发小狗反馈。
2. 视频扑克：4-6 秒，下注、保留、抽牌、结算。
3. 盲盒与装扮：4-6 秒，开盒获得道具并装备。

制作最佳实践：

- 优先录制 `1920 x 1080` 原始视频，再裁切到需要展示的区域。
- 保留真实 UI，不添加大段字幕或解释文字。
- GIF 宽度建议控制在 `600-800px`，帧率 `10-15fps`，循环无明显跳帧。
- 色彩丰富的画面会让 GIF 体积暴涨；桌宠动图可以裁切到桌面局部，扑克动图可以裁切到游戏窗口区域。
- 若 GIF 超过 `5MB`，优先缩短时长、降低尺寸、降低帧率，再考虑减少颜色。

## 竞品参考结论

### Bongo Cat

参考价值：

- 适合作为桌面伴侣、打字点击、收集帽子的受众参考。
- 不适合作为普通商店页文案模板，因为它自带 meme 认知，文案可以故意更随意。

可借鉴：

- 文案直接讲核心循环：打字/点击、收集点数、找帽子。
- 语气轻松，有角色感。

不建议照搬：

- 故意拼错和过度 meme 化。
- 过短且不解释系统的长描述。

### Deskworm

参考价值：

- 更接近 `Lucky Dog Rise` 的普通桌面伴侣竞品。
- 短描述和关于游戏都按模块清楚解释功能。

可借鉴：

- 先定义桌面宠物，再说明打字/点击触发，再讲收集和自定义。
- 关于游戏使用清晰小标题。
- 如果有概率、合成或成长系统，可以单独成段说明。

对 `Lucky Dog Rise` 的启发：

- 商店页应采用“清楚、轻松、有功能分层”的路线。
- 桌宠体验是第一卖点，视频扑克是差异点，不应让扑克压过桌面陪伴。

## 优先级

### 主人回来后优先一起处理

1. 确认最终英文短描述。
2. 确认关于此游戏的英文结构。
3. 确认是否同时填简体中文商店页文本。
4. 确认发行日期展示策略。
5. 确认内容调查中扑克与盲盒的口径。

### 后续素材优先制作

1. 一支 30 秒左右 gameplay-first 预告片。
2. 至少 5 张真实截图。
3. 一套 Steam 商店胶囊图。
4. 一套 Steam 库资产。
5. 3 个关于此游戏用的短循环 GIF。

### 当前不建议投入过多

- 高价 DLC 策略。
- Steam 库存交易化。
- 大量多语言截图。
- 复杂官方网站。
- 大量社交媒体矩阵。
