# 成就与玩家统计表说明

## 目的

`Achievement` 和 `PlayerStatistic` 共同为未来的平台成就、平台统计和数值分析提供数据定义。

- `Achievement` 定义玩家在什么条件下解锁成就。
- `PlayerStatistic` 定义游戏从第一版开始长期累计哪些数值，以及这些数值在平台 CSV 中如何识别和解释。

表格不能自行捕捉游戏行为。C# 需要在对应的真实游戏行为发生时更新统计或记录事件；表格负责让维护者配置成就条件、查询统计口径，并为未来的平台映射提供稳定名称。

Debug 的随机获得道具、发放筹码和固定牌局不应写入正式成就或玩家统计。

## Achievement

`Achievement` 是唯一的成就配置表。每一行代表一项可被玩家解锁的成就。

### 字段

- `AchievementId`
  - 成就的稳定数字 ID。
  - 发布后不得复用或改作另一项成就。

- `ApiName`
  - 成就的稳定机器名，例如 `ACH_FIRST_DOG`。
  - 未来接入 Steam 或其他平台时可作为平台成就名的基础。
  - 发布后不得随意改名。

- `RuleType`
  - 成就的判定方式。
  - 当前支持的约定如下：
    - `FirstExternalItemType`：首次从非初始来源获得指定道具部位。
    - `FirstExternalItemRarity`：首次从非初始来源获得指定品质。
    - `FirstEvent`：首次发生指定游戏事件。
    - `StatisticAtLeast`：指定玩家统计达到目标数值。

- `TargetKey`
  - 由 `RuleType` 决定含义。
  - `FirstExternalItemType` 使用 `EItemType` 枚举名，例如 `Dog`、`Headwear`。
  - `FirstExternalItemRarity` 使用 `ERarity` 枚举名，例如 `Rare`、`Epic`、`Legendary`、`Mythic`。
  - `FirstEvent` 使用稳定事件名，例如 `DogHintAsked`、`DogHintRefused`、`DesktopStarstruckEntered`。
  - `StatisticAtLeast` 使用 `PlayerStatistic.StatisticKey`，例如 `PokerChipsWon`。

- `TargetValue`
  - `StatisticAtLeast` 的数值门槛。
  - 首次类规则固定填写 `1`。

- `IsHidden`
  - 是否在未解锁前隐藏该成就。
  - 隐藏只影响展示，不影响统计和解锁。

- `Notes`
  - 记录成就意图、例外规则或维护注意事项。

### 新增成就

新增数值成就时，应优先复用已经存在的 `StatisticKey`。例如累计扑克赢取筹码达到一千万：

```text
RuleType: StatisticAtLeast
TargetKey: PokerChipsWon
TargetValue: 10000000
```

游戏加载存档或更新到含有新成就的版本时，应重新检查所有 `Achievement`。因此已经达到门槛的老玩家可以自动补解锁，不会因成就上线较晚而丢失进度。

新增首次成就时，优先使用已有的真实事件或道具部位；如果必须新增事件，先在 C# 中定义并持久化该事件的首次发生状态，再在表中填写对应的 `FirstEvent` 行。

## PlayerStatistic

`PlayerStatistic` 是玩家统计的数据字典，不是成就表。即使暂时没有成就引用某项统计，也可以保留该行，确保未来设计数值成就或分析玩家数据时拥有从第一版开始的历史累计。

### 字段

- `StatisticId`
  - 统计定义的稳定数字 ID。
  - 发布后不得复用。

- `StatisticKey`
  - 游戏内部、`Achievement.TargetKey` 和数据导出共用的稳定机器名，例如 `PokerChipsWon`。
  - C# 使用该键读取和更新对应统计。

- `DisplayName`
  - 数据导出和维护时使用的中文名称。

- `Unit`
  - 数值单位。
  - 当前使用 `Chips`、`Count` 和 `Seconds`。

- `StatisticType`
  - `Counter`：只增不减的累计数值，例如累计打字获得筹码。
  - `Maximum`：只保留最高值，例如扑克单局最高奖励。

- `PlatformApiName`
  - 未来平台统计字段的机器名，例如 `STAT_POKER_CHIPS_WON`。
  - 这不是游戏事件名。平台后台创建的字段名、平台适配器和本表必须保持一致。
  - 修改本字段不会自动修改 Steam 或其他平台后台的配置。

- `SyncToPlatform`
  - 未来接入平台统计时，是否计划同步该项数值。
  - 当前只表达配置意图；尚未接入平台时不会产生同步行为。

- `Notes`
  - 统计口径的唯一说明来源。
  - 必须写清楚包含什么、排除什么，以及计数发生的时机。

### 当前预留范围

- 游戏运行、启动次数和两种模式的时长。
- 全局打字输入次数与打字获得筹码。
- 扑克局数、输赢局数、下注筹码、赢取筹码、单局最高奖励和全部牌型次数。
- 盲盒打开、领奖和获得道具的次数。
- 开启盲盒实际花费的筹码。
- 各个常规品质与各个道具部位的非初始获得数量。
- 询问狗狗牌运，以及询问后最终获得扑克奖励的局数。

被狗狗拒绝和进入崇拜状态只作为首次事件成就记录，不累计次数。

这些统计用于未来判断成就门槛是否合理。例如可用 `PokerChipsWon` 与 `PokerModeSeconds` 评估扑克收入速度，再决定累计赢取筹码成就应设置为一百万还是一千万。

## 与平台数据的关系

游戏内的 `StatisticKey` 是跨平台的核心名称。未来接 Steam、其他平台或分析服务时，由平台适配器读取 `PlatformApiName` 和 `SyncToPlatform` 决定同步方式。

从平台下载 CSV 后，`PlayerStatistic` 可作为数据字典：通过 `PlatformApiName` 找到字段，再通过 `DisplayName`、`Unit` 和 `Notes` 理解该列的中文含义与统计口径。

平台统计只能提供已经同步的聚合数据。若未来需要分析不同玩家的收入分布、每小时收益中位数或 P90/P99，仍需要额外的匿名遥测或分析系统。

## 已废弃工作表

工作簿中保留的 `AchievementState` 因工具限制无法删除，已标记为不参与导出。

一次性状态由存档内部自动维护，例如已首次获得的非初始道具部位、已发生的首次狗狗事件和已解锁成就。维护者无需在该工作表配置数据。
