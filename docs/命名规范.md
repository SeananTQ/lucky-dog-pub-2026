# 美术资源命名规范

## 核心原则

- 全名优先，不缩写、不编码、不拼音。
- 格式：`前缀_BaseAssetName_变体_状态`
- 各部分均大驼峰（PascalCase），仅 `UI` 等无须展开的缩写保持大写。

## 前缀

前缀是资源的**大类归属**，用大驼峰，尽量完整拼写：

| 前缀 | 拼写来源 | 示例 |
|------|---------|------|
| `Button_` | Button | `Button_Primary_Hover.png` |
| `UI_` | User Interface（约定缩写） | `UI_ExpBar_Track.png` |
| `Icon_` | Icon | `Icon_Diamond.png` |
| `Card_` | Card | `Card_Base.png` |
| `Background_` | Background | `Background_MainMenu.png` |
| `Effect_` | Effect | `Effect_Glow.png` |

> `UI` 是唯一的前缀缩写。其他前缀均完整拼写。

## BaseAssetName / 变体 / 状态

PascalCase，不缩写：

```
Button_ConfirmPurchase_Default.png    ← 不是 btn_Confirm
UI_ExpBar_Track.png                   ← Track（底槽）
UI_ExpBar_Fill.png                    ← Fill（填充）
UI_ExpBar_Frame.png                   ← Frame（边框）
```

按钮三态：

```
Button_Primary_Default.png
Button_Primary_Hover.png
Button_Primary_Pressed.png
Button_Primary_Disabled.png
```

## 同一位置多资源

一个 UI 位置在游戏里会对应多张图片（如派系图标、关键词图标）。命名模式为：

```
前缀_位置名_实例ID
```

| 位置概念 | 实例ID | 示例 |
|---------|--------|------|
| 派系图标（Faction） | 具体派系 | `Icon_Faction_Humans`、`Icon_Faction_Undead` |
| 关键词图标（Keyword） | 具体关键词 | `Icon_Keyword_Charge`、`Icon_Keyword_Stealth` |
| 数值图标（Stat） | 具体数值类型 | `Icon_Stat_Attack`、`Icon_Stat_Health` |

`_Template` 作为实例 ID 保留给原型图中的示例/缺省图。

## 卡牌插画

每张卡牌一张完整插画 PNG（不裁圆角、不切头），大小卡通过引擎控制显式区域实现"推镜头"效果。

```
Card_Illustration_Rogue_Full.png     ← 完整插画
Card_Illustration_Warrior_Full.png
```

引擎侧通过 `uvRect` 或 RawImage 按需裁切，不依赖多张 PNG。圆角渲染由 Unity Mask / Shader 处理，不入图。

单图按钮（无状态切换）不需要写状态：

```
Button_Close.png
Button_Ok.png
```

## 文件夹结构

按功能模块分类，不按页面分。命名空间由文件夹路径承载，不体现在文件名中。

```
游戏资产/
  ├── Card/
  ├── UI/
  │   ├── Shop/
  │   ├── Battle/
  │   ├── Settings/
  │   ├── Common/           # 通用组件（返回、关闭、确认等跨模块共用元素）
  │   └── Icons/
  ├── Background/
  └── Effects/
```

## 与 Unity 的关系

- 工作流产出文件夹结构即 `Assets/Resources/` 内的结构，直接拷入使用。
- 加载路径即文件夹路径 + 文件名（不含扩展名）。
- 编程 AI 写代码前 `ls` 确认文件路径，然后直接写 `Resources.Load<Sprite>("路径")`。
