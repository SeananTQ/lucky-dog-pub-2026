# 多游戏模式系统说明

## 全局约定

### 术语

| 术语 | 说明 |
|------|------|
| **宿主窗口** | 唯一的 OS 级窗口，无边框、透明背景、置顶。切换模式时重设尺寸 |
| **游戏面板** | 扑克内容的渲染区，600×600 或 1200×1200 |

### 模式切换规则

- 三种模式共用一个宿主窗口，切换时重设窗口尺寸
- 按老板键（快捷键）时立即切到伪装模式，再按切回之前的模式
- 伪装模式下的模式切换按钮：伪装 → 游玩

## 桌宠伪装模式（老板来了）

又称 Boss Key 模式。快捷键切换到此模式后，宿主窗口缩到很小，看起来只是一个普通的桌面宠。

### 视觉布局

@AI [这里我修改了]
```
┌────────────────────────────┐
│ 💬 "别太累，适度放松下~"    │   ← 可选气泡
│             🐕            │   ← 小狗
│ 🔄  12:30 /  88.8K    ⚙   │   ← 矩形面板
└────────────────────────────┘
```

- 小狗（含穿戴装扮）趴在面板中间，纹理直接用现有素材缩放
- 面板左右两侧两个图标按钮：切换模式（🔄）、系统功能（⚙）
- 面板内显示时钟或筹码计数，二选一，在设置中切换
- 气泡默认显示预设的一些文字，盲盒金额重置时也会提醒玩家，可在设置中关闭
- 面板、气泡、小狗的尺寸固定，不以窗口缩放而二次缩放

### 技术实现

- 现有美术素材直接缩放（已在 TestDesktop 中验证 0.25× 视觉效果理想）
- 宿主窗口尺寸：约为面板外扩 8px 间距 ≈ 320×180
- 纯 Control 节点实现，不涉及 SubViewport

### 文字显示模式

| 模式 | 内容 | 示例 |
|------|------|------|
| 时钟 | 当前系统时间 | `22:30` |
| 计数 | 筹码数，超过 9999 缩写成 K | `88.8K` |

## 桌宠游玩模式

标准游玩模式。宿主窗口尺寸为游客观看到的实际画面大小。

### 布局

```
┌────────────────────────────────────────────┐
│ ┌──────────────┐ ┌──────────────────────┐  │
│ │ 信息面板      │ │ 扑克游戏面板         │  │
│ │ 240×600      │ │ 600×600             │  │
│ │              │ │                     │  │
│ │ (筹码/牌型   │ │ (已实现 MVP 的      │  │
│ │  奖励/系统   │ │  扑克内容，50% 缩放) │  │
│ │  按钮)       │ │                     │  │
│ └──────────────┘ └──────────────────────┘  │
└────────────────────────────────────────────┘
     <- 840 ->
```

- 信息面板默认在左侧，屏幕空间不够时置右侧（复用 PanelPositioner 逻辑）
- 扑克游戏面板 = 现有 MVP 内容通过 SubViewportContainer 渲染到 600×600
- 信息面板详细内容见[扑克游戏信息面板](./扑克游戏信息面板.md)

## 沉浸游玩模式

全屏窗口模式，采用完整的 1200×1200 美术素材。

- 扑克游戏面板：直接使用现有 MVP 的 1200×1200 布局，不缩放
- 牌桌和背景：九宫格横向拉伸以适配宽屏显示器（`TextureRect.StretchMode.Tile` / `Scale`）
- 系统功能面板：以**覆盖层（Overlay）**形式弹出，不跳转页面
  - 覆盖层半透明遮罩，系统面板浮在中央
  - 关闭覆盖层后无缝回到游戏画面
  - 复用现有 Overlay CanvasLayer 结构
- 播放完整 BGM + SFX
- 可通过老板键随时快速切换到伪装模式
- 窗口尺寸默认为显示器分辨率，可在设置中改为窗口模式
