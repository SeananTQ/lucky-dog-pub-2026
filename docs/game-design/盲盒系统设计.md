# 盲盒系统设计

## 设计目标

盲盒系统用于承接玩家的长期收集目标。玩家可以消耗筹码购买盲盒，并通过开盒过程获得小狗、头饰、眼镜、桌布、背景、卡背等装扮道具。

盲盒系统的核心体验不是单纯地把奖励发给玩家，而是让玩家在开盒过程中看到“盲盒品质逐步升级”的表演。玩家每次点击盲盒时，都有机会看到盲盒跳起、落下，并在落地后升级为更高品质。这个过程用于制造期待感和惊喜感。

系统设计应同时满足两个目标：

1. 玩家视角下，规则简单、反馈明确，有“这次开盒一路升级”的情绪体验。
2. 策划视角下，奖励概率稳定、品质概率可控，且后续可以迁移到服务端执行，降低作弊风险。

## 玩家视角规则

玩家购买一个盲盒后，进入盲盒开启流程。

开盒流程开始时，画面中显示一个盲盒。盲盒初始品质不一定总是普通品质，也可能直接以较高品质出现。较高初始品质用于制造“买到高级盲盒”的惊喜。

玩家可以点击盲盒推进开盒流程。每次点击后，盲盒会在原地跳跃，并在落下时展示本次是否升级。

每个盲盒最多拥有三次点击升级机会。每次升级幅度可能为 0 到 2 级：

1. 升级幅度为 0：盲盒品质不变。
2. 升级幅度为 1：盲盒提升一个品质。
3. 升级幅度为 2：盲盒连续提升两个品质。

盲盒最高品质为传说。当盲盒在三次点击结束前已经达到传说品质时，可以提前结束升级流程，并进入奖励展示阶段。

升级流程结束后，盲盒消失，画面展示最终获得的道具图标。玩家可以主动点击道具图标将其收起，也可以等待数秒后由系统自动收起。

## 品质设计

盲盒系统使用四个主要品质：

1. 普通
2. 稀有
3. 史诗
4. 传说

品质承担两个作用：

1. 表示玩家最终获得奖励的稀有度。
2. 表示开盒演出过程中盲盒当前所处的展示状态。

从玩家视角看，盲盒品质像是在点击过程中逐步变化。  
从策划视角看，最终奖励品质在开盒开始前已经确定，盲盒升级过程只是根据最终品质反推出来的演出。

## 抽奖核心规则

盲盒系统采用“先抽品质，再抽物品”的规则。

系统首先根据当前盲盒配置，抽取本次奖励品质。奖励品质决定本次最终奖励属于普通、稀有、史诗或传说。

确定奖励品质后，系统只在该品质对应的物品池中抽取具体道具。

这种设计的好处是品质概率稳定，便于策划控制，也便于后续向玩家展示概率。例如系统可以清晰地表达某个盲盒的传说概率，而不会因为单个物品权重变化导致品质概率难以解释。

## 奖池过滤规则

每个道具通过盲盒组 ID 归属于某个奖池。

当系统为某个盲盒抽取奖励时，只会从盲盒组 ID 匹配的道具中选择候选奖励。

奖池计算时需要遵循以下规则：

1. 只选择盲盒组 ID 与当前盲盒匹配的道具。
2. 只选择品质与本次奖励品质匹配的道具。
3. 剔除当前国家或地区版本中被隐藏的道具。
4. 权重小于等于 0 的道具不参与抽取。
5. 已拥有的唯一道具不会直接发放，但仍然会参与第一次权重随机。

这里的第 5 条是盲盒系统的重要规则。已拥有的唯一道具仍然参与第一次权重随机，是为了避免其他未拥有唯一道具的概率被动膨胀。

## 唯一道具不膨胀规则

如果策划将某个道具配置为唯一道具，通常意味着它是稀有收藏品，不应该因为玩家已经获得了其他唯一道具，就让剩余唯一道具变得越来越容易获得。

因此，系统采用“唯一道具不膨胀规则”。

同品质奖池第一次随机时，所有该品质道具都参与随机，包括玩家已经拥有的唯一道具。

如果第一次随机命中的是玩家未拥有的唯一道具，则玩家获得该道具。

如果第一次随机命中的是可重复获得道具，则玩家获得该道具。

如果第一次随机命中的是玩家已拥有的唯一道具，则不会再次发放这个唯一道具，而是进入同品质可重复道具池进行第二次随机。第二次随机只包含当前品质下可重复获得的道具，最终结果作为本次奖励发放给玩家。

这条规则的效果是：

1. 已拥有唯一道具的权重不会转移给其他未拥有唯一道具。
2. 已拥有唯一道具的权重会流向同品质可重复获得道具。
3. 玩家不会抽到已经拥有的唯一道具作为无效奖励。
4. 同品质中剩余唯一收藏品的稀缺性可以保持稳定。

例如传说奖池中存在多个唯一道具和多个可重复道具。当玩家已经拥有其中一个唯一道具时，该唯一道具仍然保留在第一次随机中。如果本次随机命中它，系统会把这次结果转化为同品质可重复道具，而不是提高其他传说唯一道具的命中率。

## 可重复道具的定位

可重复道具是盲盒系统中的概率缓冲池。

它们承担以下作用：

1. 让玩家在长期游玩中仍然有可重复获得的奖励。
2. 承接已拥有唯一道具的重复命中权重。
3. 防止玩家过快抽干完整图鉴。
4. 保持唯一道具的长期稀缺性。

原则上，每个可被抽取的品质都应该至少配置一个可重复获得道具。否则当玩家命中已拥有唯一道具时，系统将缺少合理的二次随机目标。

如果某个品质缺少可重复道具，应视为策划配置错误，并在开发或测试阶段暴露出来，而不是在正式环境中静默降级处理。

## 开盒演出生成规则

盲盒的实际奖励在开盒开始前已经确定。系统不会在玩家每次点击时重新决定奖励，也不会让点击结果真正影响最终奖励。

在确定最终奖励品质后，系统根据最终品质生成一段开盒演出路径。

演出路径包含：

1. 初始盲盒品质。
2. 第一次点击后的升级幅度。
3. 第二次点击后的升级幅度。
4. 第三次点击后的升级幅度。
5. 是否允许在达到传说品质后提前结束。

演出路径必须保证最终展示品质与最终奖励品质一致。

例如最终奖励品质为传说时，系统可以生成以下演出：

1. 初始为普通。
2. 第一次点击不升级。
3. 第二次点击提升两个品质，变为史诗。
4. 第三次点击提升一个品质，变为传说。

也可以生成另一种演出：

1. 初始为稀有。
2. 第一次点击不升级。
3. 第二次点击提升两个品质，变为传说。
4. 因为已经达到传说，提前进入奖励展示阶段。

从玩家视角看，这些过程像是点击带来了升级结果。  
从策划视角看，这些过程是根据最终奖励品质反推生成的表演剧本。

## 初始盲盒品质

盲盒初始品质不一定总是普通。

系统允许玩家购买盲盒后，直接看到较高品质的初始盲盒。这样可以制造“这次买到的盲盒本身就很高级”的惊喜。

初始品质不能高于最终奖励品质。否则会出现视觉表现与奖励结果不一致的问题。

例如最终奖励品质为稀有时，初始盲盒可以是普通或稀有，但不应该是史诗或传说。

## UI 表现分层

盲盒系统需要支持不同游戏模式下的开盒表现。

玩家可以在桌宠模式下开盲盒，也可以在标准扑克游戏模式下开盲盒。两种模式的开盒入口、布局空间、动画尺度和信息呈现方式可能不同。

因此，盲盒系统需要将“抽奖结果”和“开盒表现”分开。

抽奖核心只负责决定玩家获得什么奖励。  
开盒演出只负责根据结果生成盲盒升级过程。  
UI 表现只负责在当前模式中播放对应动画、接收点击、展示奖励图标，并在流程结束后收起。

这样后续即使桌宠模式和标准游戏模式拥有不同动画，也不会影响奖励概率和奖池规则。

## 前后端分离原则

盲盒系统需要从一开始就按照可迁移到服务端的方式设计。

当前版本可以由本地程序计算奖励结果，但抽奖核心不应与 UI 动画强绑定。

未来如果需要迁移到服务器，服务器只需要负责返回本次开盒结果，包括最终奖励、最终品质和演出路径。客户端继续负责播放动画和发放展示。

这种分离方式可以降低后续改造成本，也可以减少玩家通过修改本地逻辑影响抽奖结果的空间。

## 异常与配置要求

盲盒系统应在开发和测试阶段尽早暴露配置问题。

以下情况应视为配置错误：

1. 某个盲盒没有可抽品质配置。
2. 某个品质概率命中后，找不到对应品质的奖池道具。
3. 某个品质中存在已拥有唯一道具，但没有任何同品质可重复道具承接二次随机。
4. 所有候选道具都因为国家隐藏规则或权重配置被排除。

正式环境中的兜底策略需要在中断保护和异常处理规则中进一步确认。

## 中断保护

盲盒开启流程存在中断风险。例如玩家在开盒过程中关闭游戏、切换模式、窗口异常退出，或者未来网络请求失败。

中断保护的核心原则是“不穿帮”。玩家在看到最终奖励之前，不应该从背包、New 标记、装备状态、图鉴或其他界面提前得知最终奖励是什么。

因此，盲盒购买后可以立刻生成最终结果，但最终奖励在揭晓前只属于盲盒流程内部数据，不属于玩家正式背包。

推荐流程为：

1. 玩家购买盲盒时，系统立刻扣除筹码。
2. 系统生成最终奖励和开盒演出路径。
3. 系统保存一条未完成盲盒记录。
4. 玩家继续点击盲盒，推进开盒演出。
5. 当最终奖励图标首次展示时，奖励正式进入背包，并显示 New 标记。
6. 玩家点击奖励图标或等待自动收起后，开盒流程结束。

如果玩家在开盒过程中中断游戏，下次进入游戏后，该未完成盲盒仍然保留。

未完成盲盒会占用开盲盒入口。玩家再次点击开盲盒按钮时，系统不会购买或开启新的盲盒，而是继续打开之前中断的盲盒流程。

如果玩家中断时尚未看到最终奖励，则恢复到中断前的开盒进度，让玩家继续完成剩余点击和升级表演。

如果玩家中断时已经看到最终奖励图标，则该奖励已经正式入账。再次进入时可以恢复奖励展示界面，也可以视 UI 需要直接结束开盒流程，但不能丢失奖励。

中断保护需要保存以下信息：

1. 盲盒 ID。
2. 最终奖励 ID。
3. 最终奖励品质。
4. 开盒演出路径。
5. 当前演出步骤。
6. 是否已经展示最终奖励。
7. 是否已经正式入账。

其中最终奖励 ID 只用于恢复开盒流程。在最终奖励展示之前，它不能被背包系统当作已拥有道具处理。

## 后续需要补充的配置

为了让盲盒系统完整数据驱动，后续至少需要补充以下配置：

1. 盲盒基础配置：盲盒 ID、名称、所属奖池、购买消耗、最大点击次数、奖励展示自动收起时间。
2. 品质概率配置：不同盲盒对应普通、稀有、史诗、传说的概率权重。
3. 开盒演出路径配置：最终品质、初始品质、每次点击升级幅度、路径权重、是否允许提前结束。
4. 道具奖池配置：道具所属盲盒组、道具品质、盲盒权重、是否唯一、国家隐藏规则。

其中道具奖池配置可以继续复用 Item 表中的相关字段。盲盒基础配置、品质概率配置和开盒演出路径配置建议拆成独立表，方便策划单独调整。
