## 概述 

为了解决展开面板时，调整位置会有闪烁的问题，我决定换个思路。

如果一开始，每种模式就有一个初始的、富余的宿主窗口大小（就是每种模式能够展开的其他面板其实是上限的），这样在展开其他面板的时候就不需要调整这些面板在宿主窗口中的位置了。

只有在切换模式的时候，才重新设置宿主窗口大小，这时即使闪了，玩家应该也不在意 ？


但每种模式的初始富余宿主窗口尺寸计算方法不同，因此要在本文档中写清楚。


## 桌宠伪装模式

注意，`Z`表示探出屏幕外的区域，`_`表示透明。


### 面板清单

A. 桌宠面板
B. 测试设置面板

### 方案说明

B会出现的位置：
```
BBB
BAB
BBB
```

位置编号：
```
789
4A6
123
```
如上图所示，需要预留出A周围8个方位的B的空间。
因为用户会把A拖拽到windows屏幕的任意位置，当：
- 默认状态下，A出现在TaskBar处，B出现在A的左侧（4）靠上一点的位置（为了不超出屏幕）
- 如果用户把A移动到屏幕最左侧，导致B超出了屏幕，此时B会调整只A的右侧（6）同样也是会靠上一点
- 如果用户把A移动到屏幕最上面，则B出现在2的位置

## 桌宠游戏模式

### 面板清单

A. 标准游戏面板
B. 测试设置面板
C. 游戏信息面板

### 方案说明

默认位置：
```
_B_
CA_
___
```

当A在屏幕右下角时:
```
B_Z
CAZ
ZZZ
```

注意，B并不是在A的正九宫左上角那宫里，而是B的与A右对齐。


当A在屏幕左上角时：
```
ZZZ
ZAC
Z_B
```
注意，B并不是在A的正九宫游侠角那宫里，而是B的与A左对齐。

## 沉浸模式

### 面板清单



A. 大尺寸游戏面板
B. 测试设置面板
C. 游戏信息面板

### 方案说明

默认位置：
```
CA
```

因为沉浸模式是窗口化全屏模式，所以C固定在A左侧，其他面板出现时都是覆盖在A和C前面。
