## 概述

MVP版本基本算是结束了，（虽然还有一些细节上的小问题）为了不做重复的无用功，我们开始开发桌宠需要的实际功能。

我要实现一个“智能浮层设置面板”。

目前已经实现的这部分被视为扑克游戏界面。


### 核心目标

- 主游戏窗口：无边框、透明背景、置顶，用户可以拖拽移动
- 主窗口内有一个“设置”按钮
- 点击按钮后，在主窗口**外部**（紧贴主窗口的某一边）显示一个设置面板
- 设置面板里只有一个功能：音效开关（开启/关闭，状态保存到本地）
- 这个面板**看起来像是一个独立窗口**，但它**不是真正的独立系统窗口**，而是主窗口内的 UI 控件

### 面板的智能定位规则（重点）

面板不能出现在扑克窗口内部，必须出现在外部。

每次主窗口位置变化时（用户拖拽结束），重新计算面板位置并移动它。

### 面板的视觉与交互

- 有背景色（半透黑）和边框
- 有关闭按钮[X]，点击关闭面板
- 点击主窗口的“设置”按钮时，如果面板已打开，就关闭它；如果已关闭，就打开并重新计算位置
- 面板**不能**拖拽移动（位置由程序完全控制）

### 音效设置功能

- 一个 CheckButton 或 Toggle：“音效开/关”
- 状态保存到 `user://settings.cfg`
- 主游戏未来任何音效都需判断这个状态

### 技术约束

- 主窗口必须是透明、无边框、置顶
- 面板用 `Control` 或 `Panel` 实现，应该是不需要用 `Window` 节点，这一点后续可以再考虑。
