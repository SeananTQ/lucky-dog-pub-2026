# 打字统计功能

## 概述

玩家启动游戏后，不论焦点窗口是否在游戏上，只要按键盘或点鼠标，就会触发该功能——每触发一次筹码 +1。

## 防刷策略

| 规则 | 说明 |
|------|------|
| 按住不放 | 视为**1 次**（按键重复触发间隔内只计一次） |
| 自动脚本 | 不主动拦截 AutoHotkey 等工具，但受上限限制 |
| **每分钟上限** | **1200 筹码/分钟**（约每秒 20 次） |
| 上限溢出 | 超出上限的触发次数直接丢弃，不计入筹码 |

上限机制：
- 每秒最多累积 20 次（1200 ÷ 60 = 20）
- 用令牌桶或滑动窗口实现，超出部分静默丢弃，不给玩家负面反馈

## 生效模式

三种游戏模式均生效：
- **伪装模式**：窗口小且无焦点，是筹码增长的主要场景
- **游玩模式**：有焦点时由 Godot 输入系统处理，无焦点时由钩子处理
- **沉浸模式**：同上

## UI 反馈

- 模式不受影响，仅筹码数跳动
- 伪装模式面板的筹码计数实时更新

## 技术实现

### 原理

采用 Windows 低层键盘/鼠标钩子（`WH_KEYBOARD_LL` / `WH_MOUSE_LL`），与 Steam 游戏 Bongo Cat 相同的技术路径。

```
键盘/鼠标事件 → Windows 消息队列
    ↓
WH_KEYBOARD_LL / WH_MOUSE_LL 全局钩子（threadId=0）
    ↓
钩子回调运行在系统线程上：
  → 过滤重复按键（按住不算）
  → Interlocked.Increment 原子计数（不碰 Godot API）
  → 立刻返回
    ↓
Godot 主线程 _Process（~60fps）：
  → 读取原子计数
  → 检查每分钟上限
  → 累加筹码
  → 更新 UI
```

### 为什么用 Windows 钩子而不是 Godot 输入监听

| 方案 | 窗口有焦点 | 窗口无焦点 |
|------|-----------|-----------|
| Godot `_Input` | ✅ 正常 | ❌ 收不到事件 |
| `WH_KEYBOARD_LL` 全局钩子 | ✅ 正常（可配合 Godot 输入） | ✅ 仍能收到事件 |

### 钩子生命周期

```
Game启动 → GlobalInputTracker._Ready()
            ↓
          SetWindowsHookEx(WH_KEYBOARD_LL, callback, null, 0)
          SetWindowsHookEx(WH_MOUSE_LL, callback, null, 0)
            ↓
          运行中 → 回调累积到原子计数器
            ↓
Game退出 → UnhookWindowsHookEx()
```

### C# 实现示意

```csharp
public partial class GlobalInputTracker : Node
{
    private IntPtr _keyboardHook;
    private IntPtr _mouseHook;
    private int _pendingPresses;

    // 在系统线程上触发的回调，绝不能碰 Godot API
    private IntPtr HookCallback(int nCode, IntPtr wParam, IntPtr lParam)
    {
        if (nCode >= 0)
        {
            // 按住键盘只计 1 次：检查 LLKHF_UP 标志
            // （按住时 WM_KEYDOWN 不夹带 UP 标志，松开时才有）
            if ((lParam & LLKHF_UP) == 0) // 仅按键按下（含首次+重复）
            {
                Interlocked.Increment(ref _pendingPresses);
            }
        }
        return CallNextHookEx(IntPtr.Zero, nCode, wParam, lParam);
    }

    public override void _Process(double delta)
    {
        var count = Interlocked.Exchange(ref _pendingPresses, 0);
        if (count <= 0) return;

        // 上限检查：每分钟最多 1200
        float rate = 1200f / 60f; // ≈ 20/s
        int maxThisFrame = Mathf.CeilToInt(rate * (float)delta);
        int actual = Mathf.Min(count, maxThisFrame);

        GameManager.Chips += actual;
    }
}
```

### 注意事项

- 钩子回调运行在**系统线程**，操作 Godot 对象会直接崩溃
- 必须通过 `Interlocked` / `Volatile` 与主线程通信
- `SetWindowsHookEx` 需要 `threadId=0`（全局）才能捕获无焦点时的输入
- 程序退出前必须 `UnhookWindowsHookEx`，否则系统会残留钩子

## 验证

1. 启动游戏 → 切换到伪装模式 → 窗口失去焦点 → 按键盘 → 筹码增加
2. 按住一个键不放 → 筹码只计 1 次
3. 疯狂按键 1 分钟 → 筹码最多增加 1200
4. 切换到游玩模式 → 窗口获取焦点 → Godot 输入正常，筹码仍通过钩子累积
