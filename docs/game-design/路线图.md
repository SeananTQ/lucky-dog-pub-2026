## 路线图定位

本路线图用于回答两个问题：

- 当前项目走到哪个阶段了。
- 下一步最应该做什么。

路线图按上线阶段组织，而不是按功能类型组织。每个阶段都会混合游戏功能、Steam 准备、宣发素材、测试、版本管理和风险控制。由于项目主要由主人独立开发，同一阶段内允许穿插不同类型任务，用来调节开发节奏；但不建议跨阶段提前堆太多大功能。

首发目标是做出一个“小而完整”的 Steam 版本：

- 玩家每天打开桌宠小狗。
- 玩家通过打字和点击获得筹码。
- 小狗通过动作、表情和装扮反馈玩家状态。
- 玩家使用筹码游玩类电视扑克玩法。
- 玩家通过装扮、盲盒和成长目标获得继续打开游戏的理由。

## 反破解与反作弊策略

首发版本不建议投入重反作弊。

原因：

- 首发版本是单人桌宠 + 单人扑克，不存在多人竞技公平问题。
- 筹码和装扮如果主要保存在本地，玩家修改本地数据不会直接破坏其他玩家体验。
- 过早引入复杂反作弊可能干扰全局输入、透明窗口、置顶、点击穿透等桌宠核心技术。
- 复杂反作弊会显著增加开发和测试成本。

首发前建议只做基础防护：

- [ ] Steam 启动和所有权检查。
- [ ] 存档基础校验。
- [ ] 调试入口摘除。
- [ ] 不把开发命令暴露给正式玩家。
- [ ] 不在正式构建中保留可直接改筹码或解锁道具的入口。

需要认真做反作弊的阶段：

- [ ] 加入排行榜并影响奖励。
- [ ] 加入联网经济。
- [ ] 加入多人德州扑克。
- [ ] 筹码或道具具备现实价值兑换。

## 暂不进入首发范围

以下内容暂不进入首发范围，除非后续路线图重新评估：

- [ ] 好友系统。
- [ ] 多人房间。
- [ ] 多人德州扑克。
- [ ] 语音聊天。
- [ ] Steam Inventory。
- [ ] Steam Workshop。
- [ ] 排行榜经济系统。
- [ ] 严肃反作弊。
- [ ] 玩家自定义脚本或控制台。
- [ ] 创意工坊式开放内容。

这些内容不是不重要，而是会显著扩大首发风险。首发阶段应优先证明 Lucky Dog Pub 的核心差异：小狗桌宠、日常输入反馈、装扮成长和电视扑克轻玩法。

## 参考资料

- Steam Review Process：商店页和构建需要审核，商店页内容应只展示首发实际可用内容。
- Steam Release Process：Coming Soon 页面需要提前公开，商店页和构建审核通过后由开发者手动发布。
- SteamPipe Uploading：Steam 通过 Depot、Package 和 Branch 管理构建上传、测试分支和更新。
- Steam Discounting：首发折扣和后续折扣需要考虑冷却时间与长期价格预期。
