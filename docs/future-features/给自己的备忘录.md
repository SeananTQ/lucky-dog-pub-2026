我建议你现在先补 **2 张新表 + 扩展 2 张现有表**。其中 `BlindBox` 和 `Item` 是现有表；新增的是“品质概率表”和“开盒演出路径表”。

**现有表：BlindBox**

| 字段 | 建议类型 | 说明 |
|---|---:|---|
| Id | int | 盲盒/发奖来源 ID |
| Name | string | 显示名称 |
| RewardGroupId | int | 对应 Item 表里的奖池组 ID。也可以继续叫 BlindBoxId，但我建议新字段更清晰 |
| DisplayMode | enum/string | 展示方式：升级盲盒、直接展示奖励 |
| CostChips | int | 基础筹码价格。阶梯价格现在已有的话，保留现有字段 |
| MaxRevealClickCount | int | 普通盲盒点击次数，默认 3 |
| AutoCollectSeconds | float | 奖励展示后自动收起时间 |
| FixedRarity | ERarity/nullable | 活动盲盒可固定品质。普通盲盒为空 |
| UseRarityRate | bool | 是否先抽品质。普通盲盒 true，固定品质活动盲盒 false |
| IsEnabled | bool | 是否启用，方便临时关闭 |

`DisplayMode` 可以先简单约定两个值：

| 值 | 说明 |
|---|---|
| RevealUpgrade | 普通盲盒升级表演 |
| DirectReward | 活动盲盒直接展示奖品 |

**现有表：Item**

| 字段 | 建议类型 | 说明 |
|---|---:|---|
| BlindBoxId | int | 目前已有。表示物品所属奖池组 |
| ItemRarity | ERarity | 目前已有。用于品质池筛选 |
| BlindBoxWeight | int | 目前已有。用于同品质内随机 |
| IsUnique | bool | 目前已有。唯一道具不重复发放 |
| HiddenRegionFlag | EHiddenRegionFlag | 目前已有。国家隐藏过滤 |

Item 表暂时不用新增字段，除非你想把 `BlindBoxId` 改名成 `RewardGroupId`。我不建议现在改名，成本不值。

**新增表：BlindBoxRarityRate**

用于“先抽品质”。

| 字段 | 建议类型 | 说明 |
|---|---:|---|
| Id | int | 行 ID |
| BlindBoxId | int | 对应 BlindBox.Id |
| Rarity | ERarity | 本行代表的品质 |
| Weight | int | 品质权重 |
| IsEnabled | bool | 是否启用 |

例子：

| BlindBoxId | Rarity | Weight |
|---:|---|---:|
| 1001 | Common | 8000 |
| 1001 | Rare | 1500 |
| 1001 | Epic | 450 |
| 1001 | Legendary | 50 |

**新增表：BlindBoxRevealPath**

用于“根据最终品质反推升级表演”。

| 字段 | 建议类型 | 说明 |
|---|---:|---|
| Id | int | 行 ID |
| BlindBoxId | int | 对应 BlindBox.Id。填 0 可表示全局通用 |
| FinalRarity | ERarity | 最终奖励品质 |
| StartRarity | ERarity | 初始盲盒品质 |
| Step1Upgrade | int | 第一次点击升级 0/1/2 |
| Step2Upgrade | int | 第二次点击升级 0/1/2 |
| Step3Upgrade | int | 第三次点击升级 0/1/2 |
| Weight | int | 这条演出路径的权重 |
| CanEndEarly | bool | 达到最高品质后是否提前结束 |
| IsEnabled | bool | 是否启用 |

例子：

| FinalRarity | StartRarity | Step1 | Step2 | Step3 | Weight |
|---|---|---:|---:|---:|---:|
| Legendary | Common | 0 | 2 | 1 | 100 |
| Legendary | Rare | 0 | 2 | 0 | 80 |
| Epic | Common | 1 | 1 | 0 | 100 |
| Rare | Common | 0 | 1 | 0 | 100 |

**我建议暂时不要新建 RewardSource 表**

虽然从长期设计看，“发奖来源”更通用，但现阶段已有 `BlindBox` 表，而且你说里面已经有阶梯价格。那就先让 `BlindBox` 承担“发奖来源配置”的职责。

也就是说：

- 普通筹码盲盒：`BlindBox.DisplayMode = RevealUpgrade`
- 圣诞活动盲盒：`BlindBox.DisplayMode = DirectReward`
- 固定 Special1：`BlindBox.FixedRarity = Special1`
- 不走品质概率：`BlindBox.UseRarityRate = false`

这样表数量少，后续也能扩展，不会太早把系统拆复杂。
