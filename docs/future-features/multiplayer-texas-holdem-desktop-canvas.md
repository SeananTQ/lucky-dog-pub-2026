# 多人德州扑克与桌面社交牌桌远期构想

## 背景

Lucky Dog Pub 正式上线的第一个版本仍然以类“电视扑克”的单人玩法为主。这个方向更适合当前阶段：规则清晰、节奏轻、开发范围可控，也能和桌宠模式形成互补。

单人德州扑克缺少下注、跟注、弃牌、读牌等人与人之间的博弈，因此不适合作为第一版的核心玩法。当前版本更重要的目标，是让玩家愿意每天打开桌宠，让小狗通过打字统计、装扮、反应动作和扑克奖励逐渐建立陪伴感。

如果第一版上线后的玩家数据表现良好，后续可以考虑与合作方推进多人德州扑克玩法。届时，玩家不只是操作一局牌，而是带着自己的小狗形象进入同一张桌子，进行社交化的德州扑克体验。

## 阶段定位

第一阶段仍然围绕“单只小狗 + 桌宠陪伴 + 电视扑克”展开：

- 玩家日常开着小狗桌宠。
- 玩家通过打字、点击、游戏结算等行为获得筹码。
- 玩家使用筹码体验电视扑克式的轻量玩法。
- 玩家通过道具、皮肤、表情和反应系统形成自己的小狗形象。

远期阶段再考虑“多只小狗 + 共享牌桌 + 德州扑克社交”：

- 多名玩家进入同一房间。
- 每名玩家的小狗形象都显示在牌桌上。
- 桌面上同时显示公共牌、筹码池、座位信息和多只小狗。
- 玩家通过下注、跟注、加注、弃牌等行为进行完整德州扑克博弈。
- 小狗外观、动作和表情成为玩家的社交身份。

## 产品愿景

远期多人德州扑克并不是简单增加一个联网牌局，而是把“桌宠”与“牌桌社交”结合起来。

玩家在第一版中养成的小狗，可以在多人桌面牌局中继续出现。小狗的品种、帽子、眼镜、装饰、表情和动作都可以成为玩家的个人展示。这样，玩家进入房间时带来的不是一个抽象头像，而是一只已经有陪伴感和个性的小狗。

这个方向的核心价值包括：

- 桌宠资产可以延续到多人玩法中，降低玩家从单人内容迁移到多人内容的割裂感。
- 小狗外观可以承载社交表达，比普通头像更有记忆点。
- 多只小狗围绕牌桌互动，能形成更强的直播观赏性和截图传播性。
- 德州扑克天然适合多人博弈，可以补足第一版单人电视扑克缺少社交对抗的问题。

## 桌面画布方向

当前桌宠模式采用“胖窗口”方案，适合显示单只小狗、系统面板和轻量桌宠交互。这个方案对第一版是合理的，因为显示区域小、性能负担低，也便于处理点击穿透和置顶逻辑。

多人德州扑克阶段可能需要另一种桌面承载方式：接近全屏大小的透明桌面画布。

原因是多人牌桌需要同时容纳更多内容：

- 多只小狗。
- 公共牌区域。
- 筹码池。
- 玩家座位。
- 操作按钮。
- 表情、气泡或临时反馈动画。

如果这些内容仍然强行塞进胖窗口，窗口边界、拖拽范围、置顶逻辑和点击区域都会变得复杂。全屏透明桌面画布虽然更粗暴，但可以让多只小狗和牌桌元素在整个桌面范围内自由布局。

因此，远期可以把窗口承载方式抽象为两类：

- 胖窗口模式：服务第一版桌宠和单人电视扑克。
- 桌面画布模式：服务多人房间、多只小狗和共享牌桌。

## 核心功能设想

多人德州扑克远期版本可以包含以下能力：

- 房间系统：玩家创建房间、加入房间、邀请其他玩家。
- 座位系统：玩家选择或自动分配座位。
- 公共牌区域：显示翻牌、转牌、河牌。
- 手牌区域：只向对应玩家显示自己的手牌。
- 筹码池：显示底池、边池和玩家下注额。
- 操作流程：支持下注、跟注、加注、过牌、弃牌。
- 结算流程：展示赢家、牌型、筹码变化。
- 小狗展示：显示每名玩家的小狗形象和装扮。
- 小狗反应：根据牌局事件触发表情、动作或装饰状态。
- 社交反馈：支持简单表情、气泡、动作或预设短语。

## 可复用基础

当前版本已经在做的部分数据驱动工作，可以为远期多人玩法打基础：

- 小狗皮肤和部件由表格驱动。
- 装备系统可以表达每名玩家的小狗外观。
- `DogReaction` 可以表达小狗在不同事件下的表情反应。
- 桌宠模式已经验证了置顶、透明窗口、点击穿透、全屏应用兼容等桌面技术。
- 打字统计和活动状态可以作为桌宠陪伴玩法的长期留存入口。

远期多人玩法不应推翻这些系统，而应尽量复用这些已经被验证过的资产和数据结构。

## 技术方向

远期多人德州扑克会引入当前版本没有的复杂度，主要包括：

- 联网同步：房间状态、玩家动作、牌桌状态需要保持一致。
- 服务端权威：发牌、洗牌、结算和筹码变化应由服务端控制。
- 随机公平：牌组随机需要可审计，避免玩家怀疑作弊。
- 延迟处理：下注倒计时、断线重连、超时托管都需要设计。
- 桌面窗口策略：需要在胖窗口和全屏桌面画布之间建立清晰边界。
- 性能控制：多只小狗、动画、透明窗口和置顶逻辑需要避免明显增加玩家电脑负担。
- 隐私与兼容：桌宠覆盖层不能干扰玩家正常使用电脑，也要处理全屏游戏、任务栏和系统菜单等场景。

## 商业与合作前提

多人德州扑克涉及更重的系统建设，可能需要合作方参与：

- 账号系统。
- 房间服务器。
- 网络同步。
- 风控与反作弊。
- 社交关系链。
- 运营活动。
- 合规评估。

第一版上线后的玩家数据，是判断是否推进该方向的重要依据。只有当桌宠留存、扑克玩法参与度、装扮系统吸引力等数据达到预期时，才值得把多人德州扑克作为正式路线推进。

## 风险

远期方案存在以下风险：

- 德州扑克的规则和体验复杂度显著高于电视扑克。
- 多人联网会引入服务器成本和维护成本。
- 社交玩法需要考虑玩家行为管理和内容安全。
- 多只小狗和全屏桌面画布可能带来性能压力。
- 如果涉及具有现实价值的筹码或兑换机制，会产生额外合规风险。
- 合作方资源、排期和目标可能影响功能边界。
