# 卡牌 名称和技能抽象转Emoji

## 功能说明
将游戏中的卡牌，从“包装名+技能描述”抽象转化为：
1. **说人话**：一个直白的叫法
2. **Emoji 序列**：2~3个（最长不超过4个）emoji 组成的视觉标识

## 核心原则

### 1. 说人话的命名规范
- **剥掉所有包装**：无视原版世界观名称（如“瑟塔，万物的施恩者”），只看它实际是干嘛的
- **职业+特征**：用最通俗的词汇组合，如“和尚老师”“拾荒法师”“卖货大姐”
- **追求传神**：可以用带点调侃或民间感的叫法，比“回收型法师”更好的是“拾荒法师”

### 2. Emoji 组合规范
- **长度**：2~3个为主，最长不超过4个
- **叙事性**：不是随意堆砌，而是像一个小分镜，讲清楚这个角色是谁、在干嘛
- **固定结构**（推荐）：

| 位置 | 含义 | 示例 |
|:---|:---|:---|
| 第1个 | 角色主体（谁） | 🧙法师 / 👨‍🦲僧侣 / 👥刺客 / 🤖机器人 |
| 第2个 | 属性/环境/状态（什么） | ♻️回收 / 🌑暗影 / 🌿自然 / 💜邪能 |
| 第3个 | 动作/结果/产出（干嘛） | 🎴卡牌 / 🗡️武器 / 💰金币 / 🔮附魔 |

- **备选方案**：当角色主体+动作已经足够表达时，2个 emoji 也可以：
  - ⚒️✨ = 附魔铁匠（铁匠+附魔光芒，省略产出物也能懂）
  - 💪🔨 = 大块头战士（猛男+重武器，直击本质）

## 转化流程

1. **审视原卡**：获取包装名和技能效果
2. **提炼本质**：忽略所有玄幻名词，问自己“这张牌到底在干嘛？”
3. **命名说人话**：用 `[职业/身份] + [核心特征/动作]` 的格式命名
4. **组装 Emoji**：按“谁→什么氛围→在干嘛”的顺序挑选 emoji
